1. What Are Local Storage and Session Storage?

Both localStorage and sessionStorage are part of the Web Storage API. Unlike cookies, values stored here are:

  • Not automatically sent with HTTP requests
  • Accessible only via JavaScript in the browser
  • Ideal for client-only data such as preferences and UI state
Key differences at a glance:
Feature localStorage sessionStorage
Persistence Persists after browser is closed Cleared when tab or browser closes
Typical use Theme, layout, user settings Temporary selections, in-progress forms
Access Via global objects localStorage and sessionStorage

4. Storing and Retrieving Non-String Data

Since Web Storage only supports strings, you must convert objects and arrays using JSON.stringify() and JSON.parse().

Example Â· Object via JSON 
// Storing an object
const user = { name: "Alice", age: 25 };
localStorage.setItem("user", JSON.stringify(user));

// Retrieving the object
const storedUser = JSON.parse(localStorage.getItem("user"));
console.log(storedUser.name); // "Alice"
